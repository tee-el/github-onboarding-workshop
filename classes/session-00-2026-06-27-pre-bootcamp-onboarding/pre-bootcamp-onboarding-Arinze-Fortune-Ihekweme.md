## Key Concepts I Learned

- GitHub is a platform for hosting and sharing code using Git version control
- You can use repositories to organise projects and showcase your work publicly
- Pinning repositories on your profile helps others quickly see your best or most relevant projects
- Commits and contribution history show how active you are on the platform over time
- A good GitHub profile includes a bio, location, and links so people can learn more about you

## Lab / Hands-On Work

### What I did

For this session, I already had an existing GitHub account — **Arizonal8** — so instead of creating one from scratch, I explored my profile and reviewed what I had built so far. I looked at my pinned repositories, my contribution graph, and my activity history to understand how my work is presented publicly.

### What happened / Result

My profile already has 41 repositories and 173 contributions over the last year. Some of the projects I have pinned include:

- **Configure-Role-Based-Access-Control-RBAC-with-Azure-Entra-ID** — managing access to Azure resources using RBAC and Microsoft Entra ID
- **Azure-Secure-Web-Application-Architecture-with-Firewall-and-VNet-Integration** — deploying a secure web application on Azure using ARM templates, NSGs, and Azure Firewall
- **Cloud-Security-Implementation-with-EC2-VPC-and-Monitoring** — AWS cloud security with VPC, IAM, GuardDuty, and CloudWatch
- **Azure-Cloud-Honeypot-SOC-Lab-Unauthorised-Access-Monitoring** — a Windows VM honeypot capturing unauthorised access attempts via Microsoft Sentinel
- **Configure-hybrid-identity-with-Microsoft-Entra-Connect** — integrating on-prem Active Directory with Microsoft Entra ID using Entra Connect
- **Lumma-Stealer-Project** — CTI forensic analysis involving PCAP investigation, MITRE ATT&CK mapping, STIX 2.1 IoCs, and OpenCTI integration

My most active months were April and May 2026, with commits spread across several repositories.

### Challenges I faced

- I wasn't sure at first how to navigate the contribution activity timeline, but filtering by year helped
- Some of my older repositories don't have README files yet, which makes them look incomplete — something to fix going forward

## My Takeaways

What I found most valuable was seeing how a GitHub profile actually reads from the outside. Having your pinned repos clearly labelled and described makes a real difference. I also noticed that regular commits — even small ones — keep the contribution graph looking active, which matters when someone is checking out your profile.

It also made me want to clean up a few repos that are missing proper documentation.
