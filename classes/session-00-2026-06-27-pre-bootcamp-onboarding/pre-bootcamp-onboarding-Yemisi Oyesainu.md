### How to create a GitHub account
1. Go to GitHub.com on your browser. You will be welcomed to this interface below, Click on Sign up
2. Choose any of the available options to create your account. You can choose to use your Google or Apple profile or start signing up with an email
   
### How to customise your profile
1. To customise your profile after creating an account on GitHub, click on your profile picture or Avatar on the top right hand side of the page. This will display a pane as in the image here
2. Next Click on Settings
3. Here you can now proceed to set up your profile as desired terms of the Accessibility, Notifications, Appearance of the interface etc

### How to create a repository on GitHub
1. Click on the Hamburger icon on the top right side of the page and to Open Menu
2. Then Select Home.
3. On the page displayed there are two options for creating a new repository. First, you can click on New on the top left-hand side of the page or Click on the  + sign on the top right-hand side of the page. See the image below:
4. On the next page, give your repository a name and a brief description of the purpose of the repository. A repository name is unique within your account. You can set the visibility of your repository to Public or Private
5. You can add a ReadMe file which actually is best practice as this will explain in full details what the repository is all about. Then click on Create Repository at the bottom of the page

### How to add a file to a repository
1. Go to your repository, on the top right hand side, Click on the  **+** sign, from the dropdown there are two options. You can choose to create a new file or upload an existing file from your local environment or your computer. To create a new file, select Create new file
2. Here you can create a folder where you will keep your new file by typing the name of the folder followed by forward slash /, see **Test documentary** folder in the image below. Also the name I give my file is **Test File 1** together with *.md* which is the extension of the file. This is best practice when creating a file on GitHub
3. Next I typed into my file using Markdown which is a supported writing language on GitHub
An Heading is indicated by hash sigh **#**
4. Click on Preview to see what you have typed if there is need for any correction before saving the changes by clicking on **Commit changes**

### How to fork a repository
1. Select the repository and click on **Fork** on the top right-hand side of the screen
2. Then click on **Create Fork** at the bottom of the next page that comes up
3. To ensure that your forked branch is up to date with the upstream(main branch), click on **Sync Fork**
