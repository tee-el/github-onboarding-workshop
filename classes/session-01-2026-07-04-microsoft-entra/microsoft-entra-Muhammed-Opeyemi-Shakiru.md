## Key Concepts I Learned

<!-- Write the main ideas covered in today's session -->

-API key authentication is a straightforward approach that allows an application to access an API by including a unique key with every request. Because these keys provide access to protected resources, they must be handled securely. Rather than embedding API keys directly in source code, they should be stored in secure services such as Azure Key Vault.

-One of the main advantages of API keys is their ease of use. An application simply attaches the API key to each API request, and the API verifies the key before either processing the request or returning an authentication or authorization error if the key is invalid.

-I learned about two common methods for securing API integrations in Microsoft 365 Copilot declarative agents: API keys and OAuth 2.0.
