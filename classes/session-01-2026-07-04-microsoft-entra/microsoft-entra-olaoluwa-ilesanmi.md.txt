## Key Concepts I Learned

<!-- Write the main ideas covered in today's session -->

* - Understanding API security 
o Before integrating an API into a declarative agent, developers must determine how the API is secured. 
o The module introduces two common authentication mechanisms: 
* API Key – A simple authentication method where a unique key is sent with each request. It is commonly used for server-to-server communication and less complex integrations. 
* OAuth 2.0 – A more secure, user-based authentication protocol that allows users to grant applications limited access to their resources without sharing their passwords. It is the preferred approach for enterprise applications. 


* Integrating API plugins 
o Developers learn how to configure API plugins so Microsoft 365 Copilot can securely communicate with external services. 
o For APIs secured with an API key, the key is configured within the plugin so requests are authenticated automatically. 
o For OAuth-secured APIs, the plugin is configured to authenticate users, obtain access tokens, and securely include those tokens in API requests.
