Key Concepts I Learned

I learned that the Cloud and AI Security Bootcamp 2026 begin with proper onboarding so participants understand the program structure, expectations, tools, and method of submission.
I learned that GitHub is an important tool for the bootcamp because participants will use it to organize notes, upload assignments, fork repositories, commit changes, and submit pull requests.
I learned that a repository is a storage space for project files, notes, lab reports, screenshots, and assignment submissions. It helps keep all bootcamp work organized.
I learned that forking means creating my own copy of the main bootcamp repository. This allows me to work on my assignment without changing the original repository directly.
I learned that a pull request is used to submit my completed work from my forked repository back to the main repository for review by the facilitators.
I learned that Microsoft Azure will be useful for the hands-on cloud security labs. It gives participants access to cloud services needed for practical learning.

My Takeaways

The most valuable thing I learned from this session is how to use GitHub for assignment submission. Before this session, GitHub looked like a platform mainly for developers, but I now understand that it is also useful for documentation, collaboration, and structured learning.

I also learned the importance of following instructions carefully. In a technical bootcamp, small mistakes such as wrong file name, wrong folder, or failure to commit changes can affect assignment submission.

Another important takeaway is that cloud and AI security require practical skills. Reading about security is not enough. I need to practice with on Microsoft Azure portal.

This session gave me a clear foundation for the bootcamp. I now understand what is expected from me and how to prepare for the next sessions.
